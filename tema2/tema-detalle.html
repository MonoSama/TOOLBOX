Normas ISO de Calidad de Software

ISO/IEC 25010

Modelo de calidad del software

ISO/IEC 25010 define ocho características de calidad para software, como la funcionalidad, fiabilidad, usabilidad, eficiencia, mantenibilidad, portabilidad, entre otras. Es clave para evaluar el desempeño y calidad de software a lo largo de su ciclo de vida.

ISO/IEC 12207

Proceso de ciclo de vida del software

ISO/IEC 12207 cubre todo el ciclo de vida del software, desde la concepción hasta la retirada del software. Incluye actividades de desarrollo, pruebas, y mantenimiento, además de la gestión de proyectos.

ISO/IEC 33000 (SPICE)

Modelo de evaluación de procesos de software

ISO/IEC 33000 es un estándar para evaluar la madurez de los procesos de desarrollo de software, basado en el modelo CMMI (Capability Maturity Model Integration). Se enfoca en la mejora continua de procesos.

ISO 9001

Gestión de la calidad

ISO 9001 es una norma internacional que proporciona los requisitos para un sistema de gestión de calidad. Asegura que los procesos de desarrollo de software sean eficientes y cumplan con los requisitos del cliente.

ISO/IEC 25012

Especificación de características de calidad de datos

ISO/IEC 25012 se enfoca en la calidad de los datos en los sistemas de software. Define características como la precisión, la completitud y la fiabilidad de los datos utilizados en el software.
